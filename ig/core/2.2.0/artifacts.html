This page provides a list of the FHIR artifacts defined as part of this implementation guide.

Structures: Resource Profiles

These define constraints on FHIR resources for systems conforming to this implementation guide.

MedComCoreCareTeam

Careteam participating in the care of a patient
MedComCoreEncounter

An interaction between a patient/citizen and healthcare provider(s) for the purpose of providing healthcare service(s) or assessing the health status of a patient/citizen.
MedComCoreOrganization

A formally recognized organizations within danish healthcare, registered with a SOR identifier issued by The Danish Health Data Authority
MedComCorePatient

Patient/citizen resource to use as subject in MedCom communication.
MedComCorePractitioner

Practitioner resource is used to describe a healthcare professional or another actor involved in citizen or patient care.
MedComCorePractitionerRole

PractitionerRole resource used to describe the role of a healthcare professional or another actor involved in citizen or patient care.

Structures: Data Type Profiles

These define constraints on FHIR data types for systems conforming to this implementation guide.

AssignedIdentifier

An UUID identifier assigned by an organisation
Danish Replacement Person Identifier (E-CPR)

A Danish replacement person identifier. This shall only be used when a patient or citizen does not have an official CPR-number.
LPR3 Identifier

Identifier holding a LPR3 identifier
MedComMessagingContactPoint

The datatype ContactPoint is specified for use in MedCom messages.

Structures: Extension Definitions

These define constraints on FHIR data types for systems conforming to this implementation guide.

ContactExtension

Extension that contains the contact information
DateTimeExtension

Date and time of this payload
IdentifierExtension

An extension holding a specific identifier datatype
PractitionerExtension

Reference to a PractitionerRole and subsequent Practitioner
SenderExtension

Reference to a more specfic sender of a message.
